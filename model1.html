Model 1: Independent GPT-2 Model (10-Second, No Chunking)

This model was trained using a dataset consisting of the first 10 seconds of each track, with no chunking applied. The primary objective was to create a simpler, more focused model that learns from the initial segment of each track, capturing the essence of the vocal-guided music generation process. This approach ensures that the model is optimized for generating specific isolated stems, such as bass or full instrumental, by focusing solely on the first 10 seconds of each sample.

The model uses a custom MusicDataset class to handle the input data, including vocal audio codes, track audio codes, and positional embeddings. The GPT-2 architecture was adapted to incorporate these positional embeddings, enabling the model to align the temporal and structural aspects of the input data during training. This training setup helps the model learn to generate the individual tracks (e.g., bass, instrumental) without any cumulative mixing, ensuring precise and isolated outputs.

Key features of this model include:

  • Training on a dataset with only the first 10 seconds of each sample, providing a focused scope for learning.
  • Cosine learning rate scheduler with warmup steps for smoother optimization.
  • AdamW optimizer for managing weight decay regularization.
  • Evaluation and checkpointing at every epoch with the best model saved for reproducibility.
  • WandB logging for tracking training metrics and progress.

This model serves as a baseline for vocal-guided music generation, providing an essential foundation for further experiments with chunking and more complex datasets.

Sample Count for Instrumental Tracks

The dataset contains the following number of samples for each instrumental track:

  • Hi-Hat: 32 samples
  • Kick: 32 samples
  • Snare: 20 samples
  • Clap: 22 samples
  • Bass: 34 samples
  • Drums: 39 samples
  • Keys: 39 samples
  • Full Instrumental: 39 samples
